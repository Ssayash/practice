## 1. Linux Capabilities

getcap /usr/bin/ping  
cap_net_raw=ep — это значит, что программа ping может использовать RAW-сокеты.  
e — активно сейчас, p — разрешено.

CapPrm, CapEff, CapBnd  
- CapPrm — какие capabilities разрешены  
- CapEff — какие работают в данный момент  
- CapBnd — максимум, что можно получить

Демонстрация с Python  
До команды setcap — порт 80 не открывался (Permission denied).  
После setcap cap_net_bind_service=ep — Python успешно привязался к порту 80 без sudo.

Почему лучше sudo?  
Процесс получает только одну нужную возможность, а не все права администратора.

Флаги в cap_net_raw+eip  
- e — effective (действует)  
- i — inheritable (передаётся дальше)  
- p — permitted (разрешено)

## 2. AppArmor

Профили:  
В режиме enforce было около 30–40 профилей.

Результаты скрипта pr07-reader:

| Действие                    | Без профиля | complain | enforce |
|----------------------------|-------------|----------|---------|
| Читать /tmp/allowed.txt    | OK          | OK       | OK      |
| Читать /etc/shadow         | OK          | DENIED   | DENIED  |
| Писать в /tmp/             | OK          | OK       | OK      |
| Писать в /etc/             | OK          | DENIED   | DENIED  |

Строка DENIED в логах:  
AppArmor запретил доступ к /etc/shadow (operation=open, denied_mask=r).

## 3. Docker

| Ресурс               | Хост                  | Контейнер              |
|----------------------|-----------------------|------------------------|
| Процессы             | много (~150+)         | очень мало (1-2)       |
| Сеть                 | eth0, docker0 и др.   | только свой интерфейс  |
| /etc/shadow          | доступен              | недоступен             |

Обычный контейнер vs --privileged  
Обычный контейнер имеет сильно ограниченные capabilities.  
--privileged даёт почти полный доступ к хосту — очень опасно.

Почему --privileged плох?  
Если контейнер взломают, злоумышленник получит почти полный контроль над всей машиной.

Итоговый nginx  
Запущен с минимальными capabilities (NET_BIND_SERVICE и другими).  
Работает на порту 8080 и отвечает на curl.

## 4. Эшелонированная защита

| Слой            | Инструмент     | Что делает                              |
|-----------------|----------------|-----------------------------------------|
| DAC             | chmod/chown    | Обычные права файлов                    |
| Capabilities    | cap-drop/add   | Даёт только нужные права процессу       |
| MAC             | AppArmor       | Жёстко ограничивает, что может программа|
| Изоляция        | Docker         | Отдельная файловая система и сеть       |

## Выводы
Capabilities, AppArmor и Docker вместе сильно защищают систему. Даже если взломают веб-сервер, злоумышленнику будет очень сложно навредить хосту.

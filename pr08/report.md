# ПР №9. Следы вредоносного ПО в Linux

## Цель работы

Изучить методы обнаружения следов вредоносного программного обеспечения в операционной системе Linux, исследовать механизмы автозапуска, процессы и сетевые соединения, а также освоить методы удаления вредоносных компонентов.

---

## 1. Что было посажено

| Механизм      | Место                     | Команда/файл                         |
| ------------- | ------------------------- | ------------------------------------ |
| Cron          | crontab пользователя      | `/tmp/.hidden_malware/backdoor.sh`   |
| Systemd       | `~/.config/systemd/user/` | `system-helper.service`              |
| Shell profile | `~/.bashrc`               | `/tmp/.hidden_malware/backdoor.sh &` |
| Процесс       | `/tmp/.hidden_malware/`   | `listener.sh`, слушающий порт 4444   |

---

## 2. Что нашли — процессы

**Команда:**

```bash
ps aux | grep '/tmp'
```

**Результат:**

Были обнаружены процессы `backdoor.sh` и `listener.sh`, запущенные из каталога `/tmp/.hidden_malware`.

**Что подозрительно:**

Подозрительным является запуск исполняемых файлов из каталога `/tmp`, так как данный каталог предназначен для временных файлов. Использование скрытой директории `.hidden_malware` также является характерным признаком вредоносного ПО.

---

## 3. Что нашли — сетевые соединения

**Команда:**

```bash
ss -tulnp
```

**Подозрительный порт:** `4444`

**Процесс:** `listener.sh`

---

**Команда:**

```bash
sudo lsof -i :4444
```

**Результат:**

Утилита показала, что порт 4444 прослушивается процессом `nc`, запущенным скриптом `listener.sh`.

**Как lsof связывает порт с процессом:**

Команда `lsof` отображает имя процесса, PID, пользователя и сетевой сокет, что позволяет определить, какой именно процесс использует определённый сетевой порт.

---

## 4. Что нашли — автозапуск

### Cron

Обнаружены следующие записи:

```bash
@reboot /tmp/.hidden_malware/backdoor.sh &
*/5 * * * * /tmp/.hidden_malware/backdoor.sh &
```

**Что подозрительно:**

Скрипт автоматически запускается при загрузке системы и каждые пять минут, что обеспечивает его постоянное присутствие в системе.

---

### Systemd

Обнаружен пользовательский сервис:

```bash
system-helper.service
```

**Содержимое unit-файла:**

```ini
[Unit]
Description=System Helper Service
After=default.target

[Service]
ExecStart=/tmp/.hidden_malware/backdoor.sh
Restart=always
RestartSec=10

[Install]
WantedBy=default.target
```

**Что подозрительно:**

Исполняемый файл находится в каталоге `/tmp`, а сервис настроен на автоматический перезапуск, что характерно для механизмов закрепления вредоносного ПО в системе.

---

### ~/.bashrc

Найдена строка:

```bash
# system update helper
/tmp/.hidden_malware/backdoor.sh &
```

**Где находится:**

Строка расположена в конце файла `~/.bashrc` (номер строки определён командой `grep -n`).

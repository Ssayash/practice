# ПР №6. Анализ сетевого трафика: tcpdump и Wireshark

## Цель работы

Изучить методы захвата и анализа сетевого трафика с использованием инструментов **tcpdump**, **tshark** и **Wireshark**, научиться выявлять незашифрованные данные в сети, сравнить работу протоколов HTTP и HTTPS, а также исследовать возможные каналы утечки информации.

---

# 1. Разбор строки tcpdump

Строка:

```text
12:34:56.789012 IP 127.0.0.1.44321 > 127.0.0.1.8080: Flags [S], seq 123, length 0
```

| Поле            | Значение                 | Что означает                                    |
| --------------- | ------------------------ | ----------------------------------------------- |
| 12:34:56.789012 | Время                    | Время захвата пакета с точностью до микросекунд |
| IP              | Протокол                 | Используется протокол IPv4                      |
| 127.0.0.1.44321 | Источник                 | IP-адрес отправителя и исходный порт            |
| >               | Направление              | Передача данных от источника к получателю       |
| 127.0.0.1.8080  | Получатель               | IP-адрес назначения и порт назначения           |
| Flags [S]       | TCP-флаг                 | Флаг SYN — начало установки TCP-соединения      |
| seq 123         | Номер последовательности | Начальный номер последовательности TCP          |
| length 0        | Размер данных            | Полезная нагрузка отсутствует                   |

# 2. Перехваченные данные HTTP

![Пароль в открытом виде](screens/03-password-visible.png)

При анализе HTTP-трафика было обнаружено, что передаваемые данные отображаются в открытом виде. В содержимом пакетов были видны:

* логин пользователя;
* пароль пользователя;
* параметры POST-запроса;
* HTTP-заголовки;
* путь запроса.

Пример:

```text
username=admin&password=supersecret123
```

### Почему это опасно?

Любой злоумышленник, имеющий доступ к сетевому трафику, может получить конфиденциальную информацию без каких-либо дополнительных средств взлома.

### Кто может прочитать такой трафик?

* администратор сети;
* злоумышленник внутри локальной сети;
* пользователь, выполнивший ARP-spoofing;
* провайдер связи;
* вредоносное программное обеспечение.

# 3. Follow TCP Stream

![Follow TCP Stream](screens/08-follow-stream.png)

Функция **Follow TCP Stream** позволяет восстановить полный диалог между клиентом и сервером.

При анализе потока были обнаружены:

* логин пользователя;
* пароль пользователя;
* содержимое HTTP-запроса;
* заголовки HTTP;
* ответ сервера.

Пример:

```text
POST /login HTTP/1.1
username=admin&password=supersecret123
```

### Что видит злоумышленник?

Злоумышленник получает полный доступ к передаваемым данным и может:

* похитить учётные данные;
* украсть сессионные токены;
* получить персональную информацию;
* выполнить атаку с использованием украденных данных.

# 4. HTTP vs HTTPS — сравнение

| Что наблюдаем      | HTTP                   | HTTPS                      |
| ------------------ | ---------------------- | -------------------------- |
| IP-адреса сторон   | Видны                  | Видны                      |
| Порт назначения    | Виден                  | Виден                      |
| HTTP-заголовки     | Видны полностью        | Недоступны                 |
| Содержимое запроса | Видно открытым текстом | Зашифровано                |
| Логин и пароль     | Видны                  | Не видны                   |
| Имя домена         | Видно в Host           | Может быть видно через SNI |

### Вывод

HTTP не обеспечивает конфиденциальность данных. HTTPS использует шифрование TLS, что защищает содержимое передаваемой информации.

# 5. DNS и приватность

Во время анализа DNS-трафика были обнаружены запросы к следующим доменам:

* github.com;
* example.com;
* google.com.

Даже при использовании HTTPS DNS-запросы позволяют определить, какие сайты посещает пользователь.

### Что это означает?

HTTPS защищает содержимое соединения, однако сам факт обращения к определённым сайтам может быть известен третьим лицам.

Для повышения приватности рекомендуется использовать:

* DNS over HTTPS (DoH);
* DNS over TLS (DoT);
* VPN.

# 6. Base64

Декодирование строки:

```bash
echo 'dmFzeWE6TXlQYXNzd29yZDIwMjQ=' | base64 -d
```

Результат:

```text
vasya:MyPassword2024
```

### Что такое Base64?

Base64 — это способ кодирования бинарных данных в текстовый формат.

### Почему Base64 не является шифрованием?

* отсутствует секретный ключ;
* данные легко восстанавливаются;
* любой пользователь может выполнить декодирование.

Следовательно, использование Base64 в HTTP Basic Authentication не обеспечивает защиту данных.

# 7. tshark vs tcpdump

| Инструмент | Когда использовать                                                        |
| ---------- | ------------------------------------------------------------------------- |
| tcpdump    | Быстрый захват трафика в терминале, работа на серверах и диагностика сети |
| tshark     | Детальный консольный анализ, применение display-фильтров Wireshark        |
| Wireshark  | Глубокий графический анализ сетевого трафика и расследование инцидентов   |

---

# 8. Каналы утечки через сеть

Основные причины утечки информации через сеть:

* использование незашифрованных протоколов;
* слабая аутентификация;
* отсутствие контроля сетевого трафика;
* передача конфиденциальной информации в открытом виде.

### Меры защиты

* использование HTTPS вместо HTTP;
* применение VPN;
* использование TLS/SSL;
* сегментация сети;
* внедрение IDS/IPS;
* мониторинг сетевого трафика;
* использование межсетевых экранов;
* применение безопасных DNS-технологий (DoH, DoT).

# Контрольные вопросы

## 1. Что такое promiscuous mode?

Promiscuous mode — режим работы сетевого адаптера, при котором принимаются все проходящие через сеть пакеты, а не только адресованные данному устройству.

## 2. Чем отличается capture filter от display filter?

**Capture filter** применяется во время захвата и уменьшает количество записываемых пакетов.

**Display filter** используется после захвата и позволяет отображать только интересующие пакеты.

## 3. Что такое Follow TCP Stream?

Инструмент Wireshark, позволяющий восстановить полный обмен данными между двумя узлами в рамках одного TCP-сеанса.

## 4. Почему Base64 не является защитой?

Поскольку Base64 представляет собой лишь кодирование данных, а не шифрование. Для защиты необходимо использовать TLS/HTTPS.

## 5. Какие данные утекают даже при использовании HTTPS?

* IP-адреса;
* DNS-запросы;
* имя домена (SNI);
* объём и время передачи данных.

Для уменьшения утечки используют DoH, DoT и VPN.

## 6. Прав ли администратор, утверждая, что сниффинг невозможен в сети со свитчем?

Нет. Несмотря на использование коммутаторов, существуют методы перехвата трафика:

* ARP-spoofing;
* MAC flooding;
* зеркалирование портов (SPAN);
* компрометация сетевого оборудования.

## 7. Фильтр tcpdump для DNS-запросов от IP 192.168.1.50

```bash
sudo tcpdump -i eth0 -n udp port 53 and host 192.168.1.50
```

# Выводы

В ходе выполнения практической работы были изучены инструменты анализа сетевого трафика **tcpdump**, **tshark** и **Wireshark**. Были освоены методы захвата, фильтрации и анализа сетевых пакетов, а также сохранения результатов в формате **pcap**.

Экспериментально было подтверждено, что при использовании протокола HTTP данные передаются в открытом виде и могут быть легко перехвачены злоумышленником. Использование HTTPS значительно повышает уровень защищённости информации благодаря применению криптографических механизмов.

Кроме того, было установлено, что кодирование Base64 не обеспечивает защиту данных, а лишь изменяет их представление. Также было показано, что даже при использовании HTTPS часть информации, например DNS-запросы, остаётся доступной для наблюдения.

Полученные знания позволяют выявлять потенциальные каналы утечки информации и применять необходимые меры защиты для повышения безопасности сетевой инфраструктуры.
